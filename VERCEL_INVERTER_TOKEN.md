# Як вказати токен API інвертора на Vercel

Токен **не повинен** потрапляти в код або в репозиторій. На Vercel його потрібно зберігати в змінних середовища.

## Кроки

1. Увійдіть у [Vercel](https://vercel.com) і відкрийте свій проєкт (dashboard-home).

2. Перейдіть у **Settings** (Налаштування) проєкту.

3. У лівому меню виберіть **Environment Variables** (Змінні середовища).

4. Додайте нову змінну:
   - **Name (ім'я):** `INVERTER_API_TOKEN`
   - **Value (значення):** ваш токен API інвертора (наприклад, той, що вказано в `API_DOCUMENTATION_invertor.md`).
   - Виберіть середовища, де вона потрібна: **Production**, **Preview**, **Development** (за бажанням).

5. Натисніть **Save**.

6. **Розгорніть проєкт заново** (Redeploy), щоб змінні середовища застосувалися.  
   У вкладці **Deployments** натисніть «…» біля останнього деплою → **Redeploy**.

Після цього API-маршрут `/api/inverter` на Vercel буде брати токен із `process.env.INVERTER_API_TOKEN` і ніколи не віддасть його у клієнт.

---

## Локальна розробка

Локально токен зручно зберігати у файлі, щоб не додавати його в `.env` (файл уже в `.gitignore`).

1. У **корені проєкту** (там, де `package.json`) створіть файл **`token-inverter.txt`**.

2. Відкрийте його і вставте **одним рядком** тільки токен, без пробілів і лапок, наприклад:
   ```
   f7a9d2e1b8c34a5d9e0f21b7a6d5c4e3a2b1c0d9e8f7a6b5c4d3e2f1a0b9c8d7
   ```

3. Збережіть файл. Файл `token-inverter.txt` додано в `.gitignore`, тому він не потрапить у git і на Vercel.

4. Запустіть додаток локально (`npm run dev`). Вкладка «Інвертор» буде брати токен із `token-inverter.txt`, якщо змінна `INVERTER_API_TOKEN` не задана в середовищі.

---

## Безпека

- Не комітьте токен у репозиторій.
- На Vercel використовуйте тільки **Environment Variables**.
- Локально — тільки `token-inverter.txt` або `.env.local` (обидва в `.gitignore`).
